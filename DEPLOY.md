# Запуск бота в Docker

## Ошибка `Token is invalid!`

Telegram отклоняет токен. Чаще всего:

1. В панели указан **не тот** токен (старый, обрезанный, с кавычками или пробелами).
2. Токен задан **не в переменной `BOT_TOKEN`**, а бот подхватил другое значение из окружения.
3. Бот уже запущен **локально на Mac** с тем же токеном (второй экземпляр иногда мешает).

### Что сделать

1. Откройте [@BotFather](https://t.me/BotFather) → ваш бот → **API Token**.
2. Скопируйте токен целиком: `1234567890:AAH...` (без пробелов и кавычек).
3. В панели хостинга (Environment / Variables) задайте:

   ```
   BOT_TOKEN=вставьте_токен_сюда
   ADMIN_IDS=ваш_telegram_id
   BOT_RUN_MODE=polling
   ```

4. **Удалите** лишние секреты с похожими значениями, если они не `BOT_TOKEN`.
5. Остановите бота на Mac (если запускали): закройте процесс `python run.py`.
6. **Redeploy** / пересоздайте контейнер.

### Проверка токена (на Mac)

```bash
curl -s "https://api.telegram.org/bot<ВАШ_ТОКЕН>/getMe"
```

В ответе должно быть `"ok":true`. Если `"ok":false` — токен неверный, возьмите новый в BotFather (**Revoke** → новый токен).

## Перезапуск контейнера

В панели: **Stop** → обновить переменные → **Start** / **Redeploy**.

Через SSH:

```bash
docker ps -a
docker stop <container_id>
docker rm <container_id>
# обновить env в панели, затем снова deploy
docker logs <container_id> --tail 50
```

Успех в логах: `Starting bot polling...`

---

## Минимальные переменные

| Переменная     | Пример              |
|----------------|---------------------|
| `BOT_TOKEN`    | от BotFather        |
| `ADMIN_IDS`    | `118532885`         |
| `BOT_RUN_MODE` | `polling`           |

Остальное подставляется по умолчанию (см. `.env.example`).
